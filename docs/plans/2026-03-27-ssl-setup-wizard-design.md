# SSL Setup Wizard Design

**Date:** 2026-03-27  
**Status:** Approved

## Overview

Add a web-based setup wizard that allows users to configure HTTPS access during initial deployment. Users choose between:
1. **Domain with Let's Encrypt** - Trusted SSL certificates for users with a domain name
2. **IP-only with self-signed cert** - HTTPS via self-signed certificate for users without a domain

## Requirements

- Initial setup only (one-time configuration, not runtime switchable)
- Web-based wizard (accessed over HTTP initially, upgrades to HTTPS after setup)
- Traefik (already in stack) handles TLS termination
- Let's Encrypt for domain-based SSL (auto-provisioned by Traefik)
- Self-signed certificates for IP-only access

## User Flow

```
Fresh Deploy:
1. User runs: docker compose up -d
2. Traefik starts in HTTP mode (port 80)
3. User accesses http://<vps-ip>
4. App detects first run → redirects to /setup
5. Setup wizard presents options:
   - "I have a domain name" → enter domain → Let's Encrypt
   - "IP address only" → self-signed certificate
6. Backend configures Traefik and restarts it
7. User redirected to HTTPS URL
```

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                        Docker Compose Stack                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────┐      ┌─────────────┐      ┌─────────────┐     │
│   │  Traefik    │ :443 │   Web       │ :80  │   API       │:8000│
│   │  (reverse   │─────►│  (frontend) │      │  (backend)  │     │
│   │   proxy)    │ :80  │             │      │             │     │
│   └─────────────┘      └─────────────┘      └─────────────┘     │
│         │                                          │             │
│         │              ┌─────────────┐             │             │
│         └─────────────►│  traefik/   │◄────────────┘             │
│           reads        │  configs    │   writes                  │
│                        └─────────────┘                           │
│                                                                  │
│   Config volume: ./deploy/traefik/                              │
│   Cert storage:  ./deploy/traefik/certs/ (self-signed)          │
│                  ./deploy/traefik/acme.json (Let's Encrypt)     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

- Traefik is the only container exposed to the internet (ports 80/443)
- API writes Traefik config files to shared volume, then restarts container
- Self-signed certs generated by API, stored in mounted volume
- Let's Encrypt certs managed by Traefik (stored in acme.json)

## Backend Components

### New API Endpoints

| Endpoint                  | Method | Purpose                            |
| ------------------------- | ------ | ---------------------------------- |
| `/api/v1/setup/ssl-status`  | GET    | Check if SSL setup is complete     |
| `/api/v1/setup/ssl`         | POST   | Submit SSL mode configuration      |

### Request Schema

```json
{
  "mode": "domain" | "ip_only",
  "domain": "example.com",  // required if mode=domain
  "email": "user@example.com"  // required if mode=domain, for Let's Encrypt
}
```

### New Services

| Component              | Responsibility                               |
| ---------------------- | -------------------------------------------- |
| `SSLSetupService`      | Orchestrates SSL setup flow                  |
| `TraefikConfigWriter`  | Generates Traefik config from templates      |
| `CertGenerator`        | Creates self-signed certs (for IP-only mode) |

### Database

New `SSLConfig` model to persist SSL setup state:
- `mode`: "domain" | "ip_only" | null (not configured)
- `domain`: string | null
- `configured_at`: datetime | null

### Traefik Config Templates

**Domain mode (Let's Encrypt) - traefik.yml:**
```yaml
entryPoints:
  web:
    address: ":80"
    http:
      redirections:
        entryPoint:
          to: websecure
  websecure:
    address: ":443"

certificatesResolvers:
  letsencrypt:
    acme:
      email: "{email}"
      storage: /letsencrypt/acme.json
      httpChallenge:
        entryPoint: web

providers:
  file:
    filename: /etc/traefik/dynamic.yml
```

**Domain mode - dynamic.yml:**
```yaml
http:
  routers:
    api:
      rule: "Host(`{domain}`) && PathPrefix(`/api`)"
      service: api
      entryPoints: [websecure]
      tls:
        certResolver: letsencrypt
    web:
      rule: "Host(`{domain}`)"
      service: web
      entryPoints: [websecure]
      tls:
        certResolver: letsencrypt

  services:
    api:
      loadBalancer:
        servers:
          - url: "http://api:8000"
    web:
      loadBalancer:
        servers:
          - url: "http://web:80"
```

**IP-only mode (self-signed) - traefik.yml:**
```yaml
entryPoints:
  web:
    address: ":80"
    http:
      redirections:
        entryPoint:
          to: websecure
  websecure:
    address: ":443"

tls:
  certificates:
    - certFile: /certs/cert.pem
      keyFile: /certs/key.pem

providers:
  file:
    filename: /etc/traefik/dynamic.yml
```

**IP-only mode - dynamic.yml:**
```yaml
http:
  routers:
    api:
      rule: "PathPrefix(`/api`)"
      service: api
      entryPoints: [websecure]
      tls: {}
    web:
      rule: "PathPrefix(`/`)"
      service: web
      entryPoints: [websecure]
      tls: {}

  services:
    api:
      loadBalancer:
        servers:
          - url: "http://api:8000"
    web:
      loadBalancer:
        servers:
          - url: "http://web:80"
```

## Frontend Components

### New Routes

| Route        | Purpose                               | Auth required |
| ------------ | ------------------------------------- | ------------- |
| `/setup/ssl` | SSL setup wizard (first run only)     | No            |

### New Components

| Component            | Purpose                                   |
| -------------------- | ----------------------------------------- |
| `SSLSetupWizard`     | Main wizard container                     |
| `AccessModeSelector` | Radio buttons for domain vs IP-only       |
| `DomainInput`        | Domain name + email input with validation |
| `SetupComplete`      | Success screen with redirect/instructions |

### Frontend Flow

1. App checks `/api/v1/setup/ssl-status` on load
2. If `ssl_configured: false` → redirect to `/setup/ssl`
3. Wizard submits to `/api/v1/setup/ssl`
4. On success → show completion message → redirect to HTTPS URL

## Error Handling

| Error Case                           | Handling                                                 |
| ------------------------------------ | -------------------------------------------------------- |
| Domain DNS not pointing to VPS       | Let's Encrypt will fail → show error with instructions   |
| Ports 80/443 blocked by firewall     | Let's Encrypt will fail → show error about opening ports |
| Traefik fails to restart             | API catches error → show error, keep HTTP mode           |
| Self-signed cert generation fails    | Show error, allow retry                                  |
| User accesses HTTP after HTTPS setup | Traefik auto-redirects to HTTPS                          |

### Traefik Restart Mechanism

1. API writes new Traefik config files
2. API calls `docker restart hypertrader-traefik` (via Docker socket)
3. If restart fails, API restores old config and reports error to user

## Docker Compose Changes

Add to traefik service:
- Port 443 exposure
- Volume mounts for certs and acme.json
- Watch for config file changes
