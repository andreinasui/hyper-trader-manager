Getting started

Start using HyperTrader Manager

Prepare your wallet, Hyperliquid account, DNS, and VPS, then install the manager and finish HTTPS setup in the browser.

Prerequisites

  • An EVM wallet installed. Rabby is recommended: Rabby wallet setup guide.
  • A Hyperliquid account. Create one with the referral link: Hyperliquid referral signup.
  • USDC deposited into Hyperliquid. The bot does not enforce a minimum, but trading with less than $1,000 can be unreliable and depends on the copied account size.
  • A DNS name pointed at your VPS. DuckDNS is a free option.
  • A VPS with at least 1 CPU and 2 GB RAM. Hetzner Cloud is a good low-cost option.
  • SSH access to the VPS with sudo permissions.

Risk note

Copy trading can lose money. Start small enough that mistakes, slippage, or copied-account behavior do not create outsized damage.

1. Create wallet and Hyperliquid account

  1. Install Rabby and create or import your EVM wallet.
  2. Open Hyperliquid and connect the wallet.
  3. Deposit USDC into Hyperliquid.
  4. Keep the wallet available. You will need account details when configuring trader instances.

2. Prepare DNS and VPS

  1. Create a DNS record for the manager, for example trader.example.com.
  2. Point the DNS A record to your VPS public IPv4 address.
  3. Open inbound ports 80 and 443 on the VPS firewall or cloud firewall.
  4. SSH into the VPS before running the installer.

Free DNS option

DuckDNS can give you a free hostname such as your-name.duckdns.org. Use that hostname in the SSL setup step.

3. Install HyperTrader Manager

Run this on the VPS. It installs required system packages, downloads the production stack, and starts the manager command.

sudo apt update
sudo apt install -y curl ca-certificates
curl -sSL https://raw.githubusercontent.com/andreinasui/hyper-trader-manager/main/scripts/install.sh | sudo bash
sudo nano /opt/hyper-trader/api.env
hyper-trader-manager start

If your VPS image already has curl and certificates installed, the first two commands are still safe to run.

4. Finish production setup

  1. Open http://your-domain in a browser.
  2. The app redirects to the SSL setup page.
  3. Enter the domain that points to your VPS and an email address for Let's Encrypt notices.
  4. Wait for HTTPS setup to finish. Certificate issuance usually takes about 60 seconds.
  5. After redirect to https://your-domain, create the first admin account.

HTTPS required

Admin setup is blocked until trusted HTTPS is configured. A public DNS name and reachable ports 80 and 443 are required.

5. Create your first trader

  1. Log in to the dashboard.
  2. Create a trader from the Traders page.
  3. Add the copied account, sizing, and risk settings.
  4. Start the trader and watch logs until the first sync is complete.

Common commands

hyper-trader-manager status
hyper-trader-manager logs api
hyper-trader-manager logs web
hyper-trader-manager update
hyper-trader-manager restore-backup <version>

Troubleshooting

  • SSL setup fails: confirm DNS points to the VPS and ports 80/443 are open.
  • Domain does not load: wait for DNS propagation, then check hyper-trader-manager status.
  • Need logs: run hyper-trader-manager logs api for backend issues.
  • Need to update: run hyper-trader-manager update.
